# 1. Introduction

## 1.1 Purpose of This Manual

This user manual provides comprehensive, step-by-step guidance for all users of the **NexusHR – AI-Enabled Enterprise HR & Workforce Intelligence Platform**. The document covers every functional module of the application, including login procedures, dashboard navigation, employee management, attendance tracking, leave management, payroll processing, performance reviews, notifications, reports, profile management, and security best practices.

The goal of this manual is to empower every user — from Super Administrators to individual Employees — to efficiently and confidently operate the NexusHR platform within their designated scope of responsibility.

## 1.2 Scope

This manual covers the following modules and functionalities of NexusHR v1.0.0:

- **Authentication & Authorization** — Login, password recovery, session management, and role-based access control (RBAC).
- **Dashboard Management** — Role-specific dashboards with real-time analytics, summary widgets, and AI-powered insights.
- **Employee Management** — Employee directory, profiles, onboarding, document management, and organizational chart.
- **Attendance Management** — Clock-in/clock-out, monthly summaries, shift management, attendance correction, and daily reports.
- **Leave Management** — Leave application, approval workflows, balance tracking, team leave calendar, and compensatory off grants.
- **Payroll Management** — Salary structures, batch payroll processing, payslip generation, analytics, and audit logs.
- **Performance Management** — Performance reviews, goal setting and tracking, rating systems, and performance trend analysis.
- **Notifications** — Real-time alerts, in-app notification center, and bulk notification management.
- **Reports & Analytics** — On-demand report generation, scheduled reports, download history, and AI-powered analytics.
- **Profile Management** — Personal profile viewing, avatar upload, document management, and contact information.
- **Security & Administration** — RBAC dashboards, permissions management, audit log viewer, and platform configuration.

## 1.3 Intended Audience

This manual is designed for the following user roles:

| Audience | Description |
|----------|-------------|
| **Super Admin** | Full platform control, tenant configuration, executive command center access |
| **HR Manager / HR Director** | Employee lifecycle management, recruitment, attendance oversight, leave approvals |
| **Manager / Team Lead** | Team management, performance reviews, leave approvals for direct reports |
| **Employee** | Self-service access to attendance, leave, payslips, and personal profile |
| **Finance Manager** | Payroll processing, salary structures, financial reports, and analytics |
| **Auditor** | Read-only access to all modules, audit logs, and compliance reports |

# 2. System Requirements

## Purpose

This section specifies the minimum hardware, software, and network requirements necessary to access and operate the NexusHR platform without performance degradation.

## 2.1 Hardware Requirements

| Component | Minimum Requirement | Recommended |
|-----------|---------------------|-------------|
| **Processor** | Intel Core i3 / AMD Ryzen 3 or equivalent | Intel Core i5 / AMD Ryzen 5 or higher |
| **RAM** | 4 GB | 8 GB or higher |
| **Storage** | 500 MB free disk space (for browser cache) | 1 GB or higher |
| **Display** | 1366 × 768 resolution | 1920 × 1080 (Full HD) or higher |
| **Input Devices** | Standard keyboard and mouse/trackpad | — |

## 2.2 Software Requirements

| Software | Minimum Version | Notes |
|----------|----------------|-------|
| **Google Chrome** | v100+ | ✅ Recommended primary browser |
| **Mozilla Firefox** | v100+ | ✅ Fully supported |
| **Microsoft Edge** | v100+ (Chromium-based) | ✅ Fully supported |
| **Apple Safari** | v15+ | ✅ Supported |
| **Operating System** | Windows 10 / macOS 12 / Ubuntu 20.04 / Android 10 / iOS 15 | Any modern OS with a supported browser |

> ⚠️ **WARNING:** Internet Explorer is **not supported**. Users on legacy browsers may experience rendering issues, missing UI elements, or complete page failures. Please upgrade to a modern, supported browser.

## 2.3 Network Requirements

| Requirement | Specification |
|-------------|---------------|
| **Internet Connectivity** | Stable broadband connection (minimum 2 Mbps) |
| **Protocol** | HTTPS (TLS 1.2 or higher) |
| **Firewall/Proxy** | Ensure the application domain is whitelisted in corporate firewall/proxy settings |
| **WebSocket Support** | Required for real-time notifications and AI Copilot chat |

> ℹ️ **NOTE:** NexusHR is a cloud-hosted web application. No client-side software installation is required. The platform includes an **offline indicator** that notifies users when the network connection is lost.

**Expected Result:** Users meeting the above requirements will experience smooth, responsive interactions with the NexusHR platform, including 3D ambient backgrounds, glassmorphic UI panels, and real-time data streaming.

## 4.4 Forgot Password / Password Recovery

### Step-by-Step Instructions

| Step | Action |
|------|--------|
| 1 | On the login page, click the **"Forgot your password?"** link located below the password field. |
| 2 | The system navigates to the **Forgot Password** page (`/forgot-password`). |
| 3 | Enter your registered **email address** in the provided field. |
| 4 | Click the **"Send Reset Link"** button. |
| 5 | A password reset email will be sent to your inbox. Check your email (including spam/junk folders). |
| 6 | Click the **reset link** in the email. The system navigates to the **Reset Password** page (`/reset-password`). |
| 7 | Enter your **new password** and **confirm the new password**. |
| 8 | Click the **"Reset Password"** button. |
| 9 | Upon success, you will be redirected to the login page. Log in with your new credentials. |

> 💡 **TIP:** Ensure your new password meets the organization's password complexity requirements (e.g., minimum 8 characters, uppercase, lowercase, number, special character).

---

## 14.1 Standard Logout

### Step-by-Step Instructions

| Step | Action |
|------|--------|
| 1 | Locate the **Logout** button in the **Sidebar** — it is positioned at the bottom of the sidebar, represented by a **LogOut icon** (→ door icon). |
| 2 | Click the **"Logout"** button. |
| 3 | The system clears your authentication token and session data from the browser. |
| 4 | You are immediately redirected to the **Login Page** (`/login`). |
| 5 | The login page loads fresh, ready for a new authentication. |

`📷 [INSERT SCREENSHOT: Sidebar bottom section with the Logout button highlighted]`

**Expected Result:** The user session is terminated. All cached session data (auth token, user state, permissions) is cleared from browser storage. Attempting to access any protected route without re-authenticating will redirect to the login page.

> ⚠️ **WARNING:** Always **log out** when using a **shared or public computer** to prevent unauthorized access to your account.

> 💡 **TIP:** NexusHR includes **Session Timeout Protection** — the system will automatically log you out after a period of inactivity and display a timeout warning before doing so.

---

## 14.2 Automatic Session Timeout

### Description

NexusHR implements automatic session timeout for enhanced security:

| Behavior | Description |
|----------|-------------|
| **Inactivity Detection** | The system monitors user activity (mouse movement, keyboard input, clicks). |
| **Warning Notification** | Before session expiry, a warning modal appears informing the user that their session is about to expire. |
| **Auto-Logout** | If no activity is detected within the grace period, the user is automatically logged out. |
| **Redirect** | Upon auto-logout, the user is redirected to the login page. |

`📷 [INSERT SCREENSHOT: Session Timeout Warning modal]`

## Purpose

## 15.1 Login Issues

| Problem | Possible Cause | Solution |
|---------|---------------|----------|

## 15.2 Dashboard Issues

| Problem | Possible Cause | Solution |

| "Access Denied" error on a module | Insufficient role permissions | Verify your user role. Contact HR or System Admin if you believe you should have access. |

# 16. Frequently Asked Questions (FAQ)

## Purpose

This section addresses the most commonly asked questions about the NexusHR platform.

---

### Q1: How do I access NexusHR?
**A:** Open any supported web browser and navigate to the NexusHR application URL provided by your organization. Enter your registered email and password to log in.

---

### Q2: What should I do if I forget my password?
**A:** Click the **"Forgot your password?"** link on the login page. Enter your registered email address to receive a password reset link. Follow the instructions in the email to set a new password.

---

### Q3: Can I change my role or permissions?
**A:** No, roles and permissions are managed by the Super Admin or System Administrator. Contact your HR department or System Administrator to request a role change.

---

### Q4: How do I clock in for the day?
**A:** Navigate to the **Attendance** module from the sidebar and click the **"Clock In"** button. Your check-in time will be recorded automatically.

---

### Q5: Can I apply for leave on the same day?
**A:** Yes, you can apply for leave with a start date of today. However, same-day leave requests may be subject to your organization's leave policy and manager approval.

---

### Q6: How many leave types are supported?
**A:** NexusHR supports four leave types:
- **Casual Leave** — For personal matters
- **Sick Leave** — For health-related absences
- **Earned Leave** — Accrued based on service tenure
- **Work From Home** — For remote working days

---

### Q7: How can I view my payslip?
**A:** Navigate to the **Payroll** module from the sidebar. The **"My Payslips"** tab displays your payslip history. Click **"Download"** to save a payslip as a PDF.

---

### Q8: What is the AI Copilot?
**A:** The AI Copilot is an intelligent assistant built into NexusHR. It can answer queries about HR policies, help navigate the platform, and provide data-driven insights. Access it from the sidebar icon.

---

### Q9: How do I use keyboard shortcuts?
**A:** Press `Ctrl + K` (or `Cmd + K` on macOS) to open the **Command Menu** for quick navigation. Press `?` to view all available keyboard shortcuts in the **Keyboard Shortcuts Modal**.

---

### Q10: Is my data secure on NexusHR?
**A:** Yes. NexusHR implements industry-standard security measures including:
- JWT-based authentication with token expiry
- HTTPS/TLS encryption for all data in transit
- Role-Based Access Control (RBAC) with granular permissions
- Session timeout protection
- Audit logging for all administrative actions
- Custom permission evaluator for fine-grained access control

---

### Q11: What browsers are supported?
**A:** NexusHR supports Google Chrome (v100+), Mozilla Firefox (v100+), Microsoft Edge (v100+, Chromium-based), and Apple Safari (v15+). Internet Explorer is **not** supported.

---

### Q12: Can I use NexusHR on my mobile device?
**A:** NexusHR is a responsive web application that adapts to different screen sizes. While it is optimized for desktop use, it can be accessed on tablets and mobile devices through a supported browser.

---

### Q13: Who can generate reports?
**A:** Reports can be generated by users with the Super Admin, Admin, HR Director, HR Executive, Finance Manager, or Auditor roles.

---

### Q14: What happens if I don't clock out?
**A:** If you forget to clock out, your attendance record for the day may be marked as incomplete. You can submit an **Attendance Correction Request** through the Attendance module to update your clock-out time.

---

### Q15: How are performance reviews initiated?
**A:** Performance reviews are initiated by HR or Admin users and follow a two-step workflow: Self-Assessment (employee) → Manager Review (reporting manager). Both parties are notified at each stage.
